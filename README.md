# 🌟 2026 유바디 2마을 여름 선교 소통방

둔포 아이들과 가족초청 예배를 드리는 **수원성교회 유바디교구 2마을 여름 선교 사역팀**을 위한 소통 및 협업 웹 서비스입니다. 

팀원들이 참가자 가정 명단을 조회하고, 사역 소개 PT를 세로 스크롤형 웹진 형태로 읽으며, 각 가정이 사진과 질문 답변을 작성하면 AI가 은혜로운 소개 페이지를 자동 완성해 줍니다. 또한 응원(하트)과 댓글 기능으로 소통할 수 있습니다.

---

## ✨ 핵심 기능

1. **로그인 및 권한 관리 (아빠 이름/전화번호 뒤 4자리):**
   - 엑셀 참가 가정 명단을 자동으로 파싱하여 로그인 정보를 자동 세팅합니다.
   - 아빠/가장 성함과 전화번호 뒤 4자리로 로그인합니다. (성인 1인 참여 가정도 본인 성함으로 로그인 가능)
   - 관리자(`정다운` / `8953`) 계정은 전체 명단 조회 및 현황 추적이 가능합니다.
2. **대상자 가정 명단 (Dashboard):**
   - 구역 필터링, 완료 상태별 필터링 및 이름 검색(아빠/엄마/자녀) 기능.
   - 총 참가 가정, 소개글 완성도, 성인/자녀 총원 통계 시각화.
3. **사역 소개 페이지 (PT Viewer):**
   - **세로 스크롤 웹페이지 (기본):** PPT 이미지들의 핵심 텍스트 내용을 파악하여 카드와 타임라인, 아이콘이 어우러진 현대적인 스크롤 웹진으로 구성. 좌측에 플로팅 목차 인덱스(Scroll-spy 탑재)가 제공되어 위치 파악 및 이동이 매우 편리합니다.
   - **오리지널 슬라이드 쇼 (전환 가능):** PPT 레이아웃을 그대로 감상할 수 있는 Keynote 스타일의 슬라이드 플레이어(자동재생, 전체화면, 모바일 스와이프, 키보드 단축키 지원).
4. **우리 가족 소개 등록 및 AI 생성:**
   - 가족 사진 업로드 및 4가지 질문지 답변 작성.
   - AI를 활용하여 입력값을 은혜롭고 따뜻한 한 편의 편지 형태의 소개글로 자동 정제 및 작성해 줍니다.
   - *듀얼 모드 지원:* `GEMINI_API_KEY` 환경변수가 등록되어 있으면 구글 Gemini LLM API를 호출하여 동적 작문하고, 없을 경우 고도화된 로컬 텍스트 빌더를 활용해 감동적인 완성 카드를 즉시 생성합니다.
5. **가족 소개 피드 (Social Board):**
   - 작성 완료된 카드들이 나열되며, 비동기(AJAX) 하트 토글로 "응원하기"를 전송할 수 있습니다.
   - 각 카드 하단에 자유롭게 댓글을 남기고 격려를 나눌 수 있는 댓글창이 지원됩니다.
6. **다크 모드 / 라이트 모드 (Theme Toggle):**
   - 우측 상단 해/달 아이콘을 눌러 테마를 전환할 수 있습니다. 사용자의 환경 선호도가 `localStorage`에 자동 보관되어 페이지 이동이나 재방문 시에도 유지됩니다.
7. **모바일 반응형 웹 디자인:**
   - 폰, 태블릿, PC 해상도에 맞춰 메뉴바가 슬라이드 아웃 햄버거 메뉴로 전환되며 그리드가 1열 레이아웃으로 유연하게 재조정됩니다.

## 🛠️ 로컬 실행 방법 (Local Run)

### 1. 패키지 설치
Python 3.9 이상이 필요합니다. 아래 명령어로 필요한 핵심 패키지(`Flask`, `openpyxl`)를 설치합니다.

```bash
pip install flask openpyxl
```

### 2. 데이터베이스 초기화 (Excel 파싱)
워크스페이스 폴더에 동봉된 `2026년 유바디 2마을 여름 사역 참가 가정.xlsx` 파일의 데이터를 기반으로 SQLite `database.db`를 최초 생성 및 파싱합니다.

```bash
python3 db_init.py
```

### 3. 애플리케이션 시작
```bash
python3 app.py
```
서버는 macOS 기본 에어플레이(AirPlay) 포트 충돌을 피하기 위해 **`5050` 포트**로 열립니다. 
브라우저에서 **`http://127.0.0.1:5050`**으로 접속하세요.

## 🐙 깃허브(GitHub) 배포 방법

이 프로젝트는 동적 Flask 서버 및 SQLite 데이터베이스를 사용하므로, 정적 사이트만 호스팅하는 GitHub Pages에서는 live 작동이 불가능합니다. 따라서 **깃허브에 코드를 업로드한 뒤, 외부 무료 호스팅 서비스(PythonAnywhere 등)를 연결하여 배포**해야 합니다.

### 1. 로컬 Git 저장소 초기화 및 첫 커밋 (로컬에 세팅 완료됨)
```bash
git init
git add .
git commit -m "Initial commit: Yubadi Summer Mission Web Service"
```
*(보안 및 임시 파일 방지를 위해 `database.db`, `__pycache__/`, `static/uploads/family_*` 등은 `.gitignore`에 자동 제외 등록되어 있습니다.)*

### 2. 깃허브 원격 저장소 푸시
본인의 GitHub에 로그인하여 새로운 저장소(New Repository)를 생성한 후 아래 명령어를 터미널에서 실행하여 코드를 업로드합니다.

```bash
git remote add origin https://github.com/사용자이름/저장소이름.git
git branch -M main
git push -u origin main
```

## 🌍 PythonAnywhere 배포 & 운영 가이드

[PythonAnywhere](https://www.pythonanywhere.com)는 깃허브 저장소를 연결해 Python/Flask 애플리케이션을 무료로 배포할 수 있으며, Render 무료 티어와 달리 **서버 파일시스템이 항상 유지되어 재배포해도 데이터가 초기화되지 않습니다.**

### 1. 최초 배포
PythonAnywhere 계정의 **Bash 콘솔**에서:
```bash
git clone https://github.com/사용자이름/저장소이름.git
cd 저장소이름
pip install -r requirements.txt
python db_init.py   # 최초 1회, DB를 처음 만들 때만 실행합니다
```
그 다음 **Web** 탭에서 새 웹앱을 추가(Manual configuration)하고, WSGI 설정 파일이 이 프로젝트의 Flask 앱(`app`)을 가리키도록 지정한 후 **Reload** 버튼을 누릅니다.

### 2. ⚠️ 이후 업데이트(코드 수정 반영) 시에는 반드시 이 순서만 따르기

운영 서버의 `database.db`와 `static/uploads/`에는 팀원들이 입력한 가족 소개글, 댓글, 응원, 업로드 사진이 저장되어 있습니다. 두 파일 모두 `.gitignore`에 등록되어 git이 추적하지 않으므로 코드 업데이트로 사라지지 않지만, **`db_init.py`는 실행할 때마다 DB를 통째로 삭제하고 엑셀 명단으로 재생성하므로, 최초 배포 이후에는 절대 다시 실행하면 안 됩니다.**

```bash
cd ~/저장소이름
cp database.db ~/database_backup_$(date +%Y%m%d_%H%M).db   # 안전 백업 (권장)
git pull origin main                                         # 코드만 최신화
# python db_init.py  <- 이 줄은 실행하지 않습니다!
```
마지막으로 PythonAnywhere **Web** 탭에서 **Reload** 버튼을 눌러야 변경된 코드가 실제로 반영됩니다.
